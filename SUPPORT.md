# GitHub Rock Support

Use the links below to get help with GitHub Rock.

## Before opening an issue

1. Update to the latest available build.
2. Confirm that **Enable Device Flow** remains enabled for the configured GitHub OAuth App.
3. Revoke the old authorization and sign in again when permissions or the OAuth Client ID have changed.
4. Check the repository's open issues for an existing report.

## Report a problem

Open a support issue:

https://github.com/Sayanthrock-Developer/GitHub-Rock/issues/new/choose

Include:

- GitHub Rock version.
- Android version and device model.
- A clear description of the expected and observed behavior.
- Reproduction steps.
- Relevant screenshots with private information removed.
- The affected repository or workflow URL when it is safe to share.

Never include OAuth access tokens, client secrets, passwords, private keys, signing keystores, or signing passwords.

## Security vulnerabilities

Do not report credential leaks or security vulnerabilities in a public issue. Follow [SECURITY.md](SECURITY.md).
